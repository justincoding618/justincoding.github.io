Heart Disease Prediction

The Heart Disease Prediction project is a project that I built along while learning Machine Learning and Data Science from Zero to Mastery. It was the first milestone project that I learned how to do while learning the nuances of Machine Learining and Data Science with Andrei Neagoie and Daniel Bourke.

The Heart Disease Prediction is a classification Machine Learning Project where you try to use Machine Learning tools to predict whether a patient has heart disease or not. I first needed to do a 4-step framework in order to predict patients with heart disease.

  • The first step is to define the problem. The heart disease problem that we’re looking at is an end-to-end classification problem. Classification is whether something is one thing or another. Specifically, this is a binary classification problem. There are only two classes: someone has heart disease and someone does not have heart disease.
  • The second step is to get the data. The project is based on the original repository through UC-Irvine. In addition, the dataset was obtained through Kaggle.
  • The thrid step is evaluating what needs to be done. The goal of the project is if the model can reach 95% accuracy at predicting whether a patient has heart disease or not, the project can be pursued.
  • The fourth step is looking at the data features. This is where you have to closely look at the data set that you're working with.

After determining the four steps, this is where I applied the four steps into practice. The tools that I used to complete the project were Python libraries seaborn, scikit-learn, pandas, numpy, and matplotlib.

  • Seaborn was used to visualize the confusion matrix. Matplotlib is used to visualize the data (such as creating line plots and histograms).
  • Scikit-learn was used to test different machine learning models. The different modesl that were tested are Logistic Regression, KNeighborsClassifier, and RandomForestClassifier. Importing the right models depends on model map as described by scikit-learn.
  • Pandas was used to read the dataset and exploring the data frame. If the dataset contained missing values, it is required to fill out those missing values.
  • Numpy was used to implement a random seed. The random seed allows us to still generate random numbers, but as reproducible random numbers. If we wanted someone else to be able to use our notebook and generate numbers but have the same random numbers as us and reproduce our experiment exactly, we would set a random seed. In short, it’s making sure the random numbers in the cell that it’s been run are the same every single time.
